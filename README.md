## PiSmartSite (SmartSite)

Plateforme **full-stack** pour la gestion de projets de construction et de maintenance : tableau de bord (projets, tâches, ressources, planning, documents, photos d’avancement), API **NestJS** et persistance **MongoDB**.

### Points forts

- Gestion des projets, tâches, ressources, budgets, documents et équipe.
- Planning **Gantt**, Kanban, rapports.
- **Documents** et **progress-photos** (validation, estimation IA côté backend selon configuration).
- API REST modulaire ; génération de tâches assistée par IA (optionnel, OpenRouter / Gemini).

## Stack du dépôt (projet)

| Couche | Technologie |
|--------|-------------|
| **Frontend** | **Next.js** (App Router) + **React**, Tailwind, Radix UI |
| **Backend** | **NestJS**, Mongoose |
| **IA chantier (optionnel)** | `smartsite-ai-service` — FastAPI, YOLO |

```
PiSmartSite/
├── smarsite-frontend/     # Next.js + React
```

## Prérequis

- Node.js 18+ (recommandé)
- npm ou pnpm
- MongoDB (local ou Atlas)

## Installation rapide

### Frontend

```bash
cd smarsite-frontend
npm install
npm run dev
```

### Backend

```bash
cd smartsite-backend
npm install
npm run start:dev
```

- Frontend : [http://localhost:3000](http://localhost:3000)  
- Backend : [http://localhost:3200](http://localhost:3200)

### Variables d’environnement

- **Frontend** : `smarsite-frontend/.env.local` — voir `.env.example` (`NEXT_PUBLIC_API_URL`, clés OpenRouter / Gemini si besoin).
- **Backend** : `smartsite-backend/.env` — voir `.env.example` (MongoDB, Groq, Telegram, etc.).

### Service IA (`smartsite-ai-service`, optionnel)

Utilisé pour l’analyse d’image sur les photos d’avancement. Sans ce service, l’API peut enregistrer la photo sans analyse détaillée.

```bash
cd smartsite-ai-service
python -m venv .venv
# Windows : .venv\Scripts\activate
pip install -r requirements.txt
set YOLO_MODEL_PATH=weights\detect.pt
uvicorn main:app --host 0.0.0.0 --port 8001
```

Dans `smartsite-backend/.env` : `AI_ANALYSIS_URL=http://127.0.0.1:8001/analyze-image` (voir `.env.example`).
